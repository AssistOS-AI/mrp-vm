The Lifecycle of an Intent (Execution Pipeline)

From Raw Language to Grounded Answer

The MRP-VM pipeline is a deterministic sequence of transformations designed to turn a messy natural language request into a verifiable result [DS002].

Phase 1: Intent Normalization and Validation

The pipeline begins with Neural Normalization [DS006]. The VM rewrites the request into Intent CNL [DS004].

## Intent Group 1
Act: compare
Intent: Compare BM25 and dense retrieval for lexical search.
Criterion: Fast response time, low memory.
Output: Comparative recommendation.

Phase 2: Intent Decomposition

Once validated, the intent is symbolically decomposed [DS011] into targets and criteria.

Phase 3: Evidence Selection

The VM executes a Retrieval Plan [DS012, DS023] across session contexts and the Knowledge Base.

Phase 4: Regulated Execution (Plugins)

Dispatches to specialized interpreters (e.g., Z3) for formal proof [DS003].

Phase 5: Synthesis and Grounding

Final response generation [DS017] under a strict grounding policy.
