The Neuro-Symbolic Virtual Machine

Architecture: The Governed Control Layer

The MRP-VM functions as a governed runtime for language-centric execution. Unlike conventional VMs that process opcodes, MRP-VM processes Intents and Context expressed in Controlled Natural Language (CNL). See DS001.

Structural Components

  1. MRPEngine — Central coordinator. 12-phase pipeline detail →
  2. NL Normalizer — Semantic bridge NL↔CNL. CNL formats →
  3. Intent Decomposer — Symbolic splitting into independent groups
  4. Retrieval Engine — BM25 + HDC/VSA multi-strategy. Algorithms detail →
  5. Plugin System — External interpreters. Plugin guide →
  6. Session State — In-memory conversation management. Session lifecycle →

Typed Plugins

Execution is organized around typed plugins: seed detectors, KB plugins, goal solvers, and planner plugins. Built-in symbolic and LLM-backed variants remain available, but they are selected as plugins rather than as global VM modes.

KB Plugins

Evidence selection is governed by KB plugins: kb-fast (BM25 only), kb-balanced (BM25 + HDC escalation), and kb-thinkingdb (bounded symbolic closure).
