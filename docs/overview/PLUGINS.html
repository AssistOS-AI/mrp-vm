Specialized Interpreters and Symbolic Grounding

The Plugin Ecosystem

In MRP-VM, a plugin is a specialized engine that processes a resolved intent bundle within a specific reasoning regime. Plugins are external processes — they can be written in any language. See DS003, DS016.

Key Concepts

  • Encapsulated Processes — plugins run as separate OS processes (Node.js, Python, Rust, etc.)
  • CNL I/O — communication via stdin/stdout using CNL Markdown
  • Post-Retrieval — plugins receive the full resolved evidence bundle, not raw user input
  • One per Intent — exactly one plugin is selected per intent group

The Wrapper Convention

wrappers/z3-solver/
├── manifest.json    # capabilities, keywords, command
├── wrapper.js       # the actual interpreter
└── README.md

Plugin Result Format

## Plugin Result
Status: success
Plugin: z3-solver
Confidence: high
Result: Constraint is satisfiable.
Evidence: Z3 returned SAT in 12ms.

Plugin results are integrated as explicit ### Plugin Evidence sections in the synthesis prompt — never merged silently with KB evidence.
