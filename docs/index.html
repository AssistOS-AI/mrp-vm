MRP-VM Documentation

MRP-VM is a thin execution kernel for typed plugins. The VM owns session state, execution frames, request-global budgets, tracing, and the AchillesAgentLib bridge.

Everything that looks like "reasoning policy" belongs to plugins: seed extraction, KB indexing, retrieval, solver choice, answer shaping, and validation. The VM glues plugin stages together and remains agnostic to the payload semantics beyond plugin-type boundaries.

The default system still speaks natural-language CNL between plugins, but the architecture is intentionally open to more formal IRs for domains such as logic or mathematics without changing the VM's orchestration role.

Responsibility Boundary

VM owns: frames, loops, budgets, session/workspace state, tracing, shared LLM roles.
Plugins own: KB indexing, retrieval policy, decomposition policy, solver heuristics, validation logic.
KU routing: phase-aware KUs can target planner, KB, solver, decomposition, or validation.
Default docs: built-in plugins have explicit DS coverage; plugin-private backends are documented at plugin level, not as VM internals.
Start with VM Execution Model for the kernel boundary, then read Plugin Development Guide for the concrete repository layout: src/core/**, src/plugins/<type>/<id>/**, and src/mrp-vm-sdk/**.
