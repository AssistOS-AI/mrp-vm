Processing Modes IMPLEMENTED

Every language-processing step (normalization, context extraction, synthesis) goes through a strategy interface (DS022). Two modes are available:

Corrective retry: if LLM output fails CNL validation, the error details are fed back to the LLM for one retry attempt. If that also fails, the request returns an explicit error.

symbolic-only IMPLEMENTED

Rule-based heuristics, no LLM calls.

Intent Normalization

Detects the pragmatic act by matching verbs in the input:

compare → ['compare','contrast','difference','versus','vs']
explain → ['explain','why','how','cause','reason']
recommend → ['recommend','suggest','advise','best']
diagnose → ['diagnose','problem','issue','debug']
implement → ['implement','build','create','setup','configure']
verify → ['verify','check','validate','prove','confirm']
define → ['define','what is','meaning','definition']
evaluate → ['evaluate','assess','measure','rate','review']

Short inputs (<200 chars) become a single Intent Group. Longer inputs are split by sentence.

Session Context Extraction

Filters sentences by verb patterns (is, are, use, have, deploy, prefer, need...). Excludes questions and commands. Assigns roles by keyword: prefer/want → Evaluation, deploy/must/should → Constraint, default → Explanation.

Answer Synthesis

Deterministic Markdown templates. Lists evidence claims as bullet points. No free-form prose generation. Status is answered if evidence exists, no-context otherwise.

Strategy Resolution

Priority order: request processing_mode → session preference → deployment default (config/strategies.json). No silent fallback — if the selected strategy can't handle the input, it returns an explicit error.
