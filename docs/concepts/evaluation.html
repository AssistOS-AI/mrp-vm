Evaluation System IMPLEMENTED

Suite-based NL evaluation with isolated servers, matrix testing across modes × profiles, and separate answer/context quality tracking. Spec: DS021.

Suite Structure

test/evaluation/
├── run.mjs              # Runner
├── suite01/
│   ├── story.nl         # Source text + questions
│   └── eval.json        # Expected intents, context, answers
└── suite02/
    ├── story.nl
    └── eval.json

eval.json Format

{
  "suiteId": "suite01",
  "title": "Lumina-7 — Single-Word Answers",
  "storyFile": "story.nl",
  "questions": [{
    "id": "s01-q01",
    "input": "Would Kaelen's neural implants help in the Frozen Abyss? One word.",
    "expected": {
      "intents": [{ "intentMustMention": ["Kaelen", "neural implants"] }],
      "contextMustMention": ["neural implants"],
      "contextMustNotMention": ["Flux Core", "Quartz Desert"],
      "answerMustMention": ["No"],
      "answerMustNotContain": ["maybe", "possibly"]
    }
  }]
}

What Gets Checked

CheckCategoryWhat it verifies
Intent matchingANSAt least one intent group mentions expected terms (order-independent)
Context recallCTXRetrieved context contains expected terms
Context precisionCTXRetrieved context does NOT contain unwanted terms
Answer contentANSAnswer contains expected terms
Answer noiseANSAnswer does NOT contain unwanted terms

Failures are tagged [ANS] or [CTX] so you can tell if the problem is synthesis or retrieval.

Matrix Evaluation

Runs all mode × profile combinations. One isolated server per suite (tmpdir with separate KB). Shared LLM cache for deterministic, cost-free re-runs.

CLI Options

node test/evaluation/run.mjs                     # all suites, all combos
node test/evaluation/run.mjs --suite suite01     # one suite
node test/evaluation/run.mjs --profile fast      # one profile
node test/evaluation/run.mjs --mode symbolic-only # one mode
node test/evaluation/run.mjs --delay 3000        # delay between combos (rate limiting)
node test/evaluation/run.mjs --port 5000         # custom base port
